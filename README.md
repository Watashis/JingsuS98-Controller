# JingsuS98-Controller (Jingsu S98 Python Library)

![Python](https://img.shields.io/badge/python-3.6+-blue.svg)

Библиотека на Python для управления механической клавиатурой **Jingsu S98** (или совместимыми) по USB (HID). Разработана путём реверс-инжиниринга протокола обмена данными с клавиатурой (захват трафика официального ПО через Wireshark + USBPcap). Позволяет управлять подсветкой, переназначать клавиши на обоих слоях (Top/Fn), читать заряд батареи и кормить встроенный музыкальный визуализатор.

## Возможности (Features)
- 🔋 **Мониторинг батареи**: получение текущего процента заряда аккумулятора.
- 🌈 **Глобальная подсветка**: единый статичный цвет для всей клавиатуры.
- 🎨 **Встроенные эффекты**: переключение режимов анимации (Breath, Spectrum, Rotating и др.) с настройкой скорости, яркости и базового цвета.
- ⌨️ **Per-Key RGB**: индивидуальная настройка цвета для каждой клавиши.
- 🔁 **Переназначение клавиш (Remap)**: смена назначения любой клавиши на обоих слоях (**Top** и **Fn**), в том числе на сочетания с модификаторами (Ctrl/Shift/Alt/Win). Запись хранится в самой клавиатуре и переживает отключение от компьютера.
- 🧹 **Сброс раскладки**: очистка одной клавиши или всего слоя целиком.
- 🎵 **Музыкальный live-режим**: передача уровней амплитуды звука встроенному в прошивку визуализатору.

## Быстрый старт

```python
from keyboard_backlight_library import KeyboardBacklight

# Vendor ID и Product ID могут отличаться, проверьте их в диспетчере устройств
kb = KeyboardBacklight(vid=0x05AC, pid=0x024F)

print(f"Заряд батареи: {kb.get_battery_level()}%")
kb.set_builtin_mode('Breath', r=255, g=0, b=0)
kb.remap_key('W', 'B')          # клавиша W будет печатать B

kb.close()
```

## Подсветка

### Заряд батареи
```python
battery = kb.get_battery_level()   # int 0-100 или None
print(f"Заряд: {battery}%")
```

### Встроенные эффекты
Доступные режимы: `Static`, `Breath`, `Spectrum`, `Glittering`, `Falling`, `Colourful`, `Outward`, `Scrolling`, `Rolling`, `Rotating`, `Explode`, `Launch`, `Ripples`, `Flowing`, `Pulsating`, `Tilt`, `Shuttle`, `Off`.

```python
# Пульсация красного (яркость 0-5, скорость 0-5)
kb.set_builtin_mode('Breath', r=255, g=0, b=0, brightness=5, speed=3)

kb.set_builtin_mode('Off')          # выключить подсветку
```

### Глобальный статичный цвет
```python
kb.set_global_color(r=0, g=255, b=0)   # вся клавиатура зелёная
```

### Per-Key RGB
> [!WARNING]
> Индивидуальная настройка клавиш пишется в EEPROM (flash-память) клавиатуры. **Не используйте этот метод для собственных анимаций (чаще 1 раза в несколько секунд)** — это изнашивает память и может подвесить контроллер.

```python
# Тусклый синий по всей клавиатуре, WASD — красным
layout = {key: (0, 0, 30) for key in kb.VALID_KEYS}
for key in ['W', 'A', 'S', 'D']:
    layout[key] = (255, 0, 0)
kb.set_per_key_colors(layout)
```

## Переназначение клавиш (Remap)

Каждая клавиша может быть переназначена отдельно на каждом из двух слоёв:
- **Top** (`fn_layer=0`) — обычный слой.
- **Fn** (`fn_layer=1`) — действие при зажатой клавише Fn.

Переназначение записывается в энергонезависимую память клавиатуры: оно **сохраняется при отключении от компьютера** и работает без какого-либо софта.

### Одна клавиша
```python
# Физическая W печатает B (на слое Top)
kb.remap_key('W', 'B')

# На слое Fn: Fn+W печатает B
kb.remap_key('W', 'B', fn_layer=1)

# Сочетание: V выводит Ctrl+S
kb.remap_key('V', 'S', modifiers=['Ctrl_L'])

# Несколько модификаторов: C выводит Ctrl+Shift+C
kb.remap_key('C', 'C', modifiers=['Ctrl_L', 'Shift_L'])
```

- `physical_key` — имя физической клавиши из `kb.KEY_MAP` (`'W'`, `'B'`, ...) или её индекс.
- `output_key` — что выводить: имя из `kb.HID_USAGE` (`'B'`, `'PrintScreen'`, ...) или числовой USB HID usage code.
- `modifiers` — список имён из `kb.MODIFIER_BITS` (`'Ctrl_L'`, `'Shift_L'`, `'Alt_L'`, `'Win_L'` и правые варианты `_R`) либо битовая маска.
- `fn_layer` — `0` (Top) или `1` (Fn).

### Целая раскладка одним вызовом
`set_full_keymap` атомарно заливает раскладку всего слоя (так же, как официальный софт загружает профиль). Клавиши, не указанные в словаре, сбрасываются в стандартное поведение.

```python
kb.set_full_keymap({
    'W': 'B',                       # простая клавиша
    'V': ('S', ['Ctrl_L']),         # сочетание: кортеж (клавиша, модификаторы)
    'CapsLock': 'Esc',
}, fn_layer=0)
```

### Сброс
```python
kb.clear_remap('W')          # вернуть одну клавишу к стандартному поведению
kb.clear_layer(fn_layer=0)   # сбросить ВЕСЬ слой Top в дефолт (рекомендуется)
kb.clear_layer(fn_layer=1)   # сбросить слой Fn
```

> [!NOTE]
> Отдельной аппаратной команды «удалить ремап» не существует — официальный софт «очищает» клавишу, переназначая её саму на себя. `clear_remap()` делает то же самое. `clear_layer()` сбрасывает весь слой одной bulk-выгрузкой пустой раскладки (предпочтительный способ — чистит любые клавиши, не только буквы).

## Музыкальный live-режим

Передаёт встроенному в прошивку визуализатору уровни амплитуды звука по 19 «полосам» (колонкам клавиш). Не пишет в EEPROM — можно слать часто.

```python
import math, time

t0 = time.time()
while time.time() - t0 < 5:
    phase = (time.time() - t0) * 4
    bands = [int((math.sin(phase + i * 0.5) + 1) / 2 * 6) for i in range(19)]
    kb.send_music_frame(bands, active=2)   # active: 0=тишина, 1=fade, 2=звук
    time.sleep(0.03)
kb.send_music_frame([0] * 19, active=0)    # погасить
```

> [!NOTE]
> Это **только фид амплитуды** — сама анимация (что и где загорается) зашита в прошивку. Произвольно «рисовать» кадры через этот канал нельзя: клавиатура сама решает, как отобразить переданные уровни.

## О слоях и «профилях»

В самой клавиатуре хранится **одна** активная раскладка (Top + Fn) в энергонезависимой памяти. «Профили» в официальном ПО — это локальные XML-файлы на диске; при выборе профиля софт **целиком заливает** его раскладку в эту единственную память клавиатуры. Поэтому:
- переназначение переживает отключение от ПК (живёт в чипе);
- открытие другого профиля **перетирает** текущую раскладку (заливается новая).

## Как найти VID и PID
- **Windows**: Диспетчер устройств → Клавиатуры → Свойства → Сведения → ИД оборудования (ищите `VID_XXXX&PID_YYYY`).
- **Linux**: команда `lsusb`.
